# Claude 指导规则：Vue3 + UniApp 前端项目

## 基本要求

- 严格按照要求执行，不添加额外功能或设计元素，不随便修改没有要求修改的模块
- 使用最新版 Vue 3 Composition API 语法，不混用 Options API
- 所有代码需满足 UniApp 跨平台开发规范
- 必须使用 TypeScript 进行类型检查

## 项目结构规范

- 按照功能模块创建目录，遵循清晰的目录结构
- 组件命名使用 PascalCase
- 页面文件命名使用 kebab-case
- 确保所有资源引用使用相对路径

## 组件开发规则

- 组件需按原应用拆分，保持功能独立性
- 使用 `<script setup>` 语法，提高代码可读性
- 避免过度封装，保持组件结构与原应用一致
- 明确定义 props 和 emits 类型

## 样式处理规范

- 使用 SCSS 预处理器，按原应用组织样式规则
- 使用 rpx 单位确保在不同设备上正确缩放
- 遵循 BEM 命名规范避免样式冲突
- 使用 CSS 变量存储颜色和间距等值

## UniApp 特定规则

- 使用条件编译处理平台差异性问题
- 仅使用 UniApp 支持的 API，不使用平台特定 API
- 避免使用不兼容的 DOM 操作
- 所有网络请求使用 uni.request 封装

## 性能优化考量

- 使用 shallowRef/shallowReactive 优化大型数据结构
- 避免深层组件嵌套和过度响应式
- 合理使用 v-if 和 v-show
- 仅在必要情况下使用计算属性和监听器

## 错误预防措施

- 实现统一错误处理机制
- 为所有异步操作添加适当的加载状态和错误状态
- 开发过程中持续测试在不同设备上的表现
- 确保用户输入数据验证和异常处理

## 错误管理与文档更新

- 记录开发过程中出现的所有错误
- 当同一类错误被纠正超过两次时，自动将该错误的规避规则更新到本文档中
- 更新的规则应包含错误描述、发生原因和具体解决方案
- 定期审查文档中的错误规则，确保它们仍然适用
